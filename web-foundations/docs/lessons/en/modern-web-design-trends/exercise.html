Phase 2

üî® Build Your Page (75 minutes)

‚è±Ô∏è Time Management: Spend roughly 25-30 minutes per trend. It's better to implement 2 trends well than 4 trends poorly .

Building Process

  1. Start with HTML Structure (10 min)
    Create semantic HTML for your content. Keep it simple initially‚Äîyou can always add more sections later.
    Suggested structure: Header with hero section, 2-4 content sections, footer. This gives you elements to style!
  2. Implement Trend #1 (25 min)
    • Open the corresponding demo file for reference
    • Copy the core CSS technique (not everything!)
    • Adapt it to your content and design
    • Test on different screen sizes
    • Ensure accessibility (motion preferences, contrast, etc.)
  3. Implement Trend #2 (25 min)
    Same process as Trend #1. Make sure it complements rather than conflicts with your first trend.
  4. Implement Trend #3 (Optional, 25 min)
    If time permits and you've completed 2 trends solidly, add a third. Otherwise, use this time to polish and refine.
  5. Polish & Refine (10 min)
    • Adjust spacing and alignment
    • Fine-tune colors and transitions
    • Test all interactive elements
    • Verify responsive behavior
    • Check browser DevTools for errors

Implementation Checklist

Use this as you work. Check off items as you complete them:

  • HTML: Semantic structure with proper headings and sections
  • CSS Organization: Well-organized styles with clear comments
  • Trend #1: Core technique implemented and working
  • Trend #2: Core technique implemented and working
  • Trend #3: (Optional) Core technique implemented and working
  • Responsiveness: Tested on mobile (320px) and desktop (1920px)
  • Accessibility: Sufficient contrast, respects reduced motion
  • Performance: Page loads quickly, animations are smooth
  • Browser Testing: Works in Chrome/Firefox/Safari
  • Code Quality: Clean, commented, follows best practices
‚ö†Ô∏è Common Pitfalls to Avoid:
  • Copying entire demo files without understanding the code
  • Ignoring mobile/responsive considerations
  • Forgetting accessibility (contrast, motion preferences)
  • Over-animating (less is more!)
  • Not testing on different screen sizes
Phase 3

üí≠ Critical Reflection (30 minutes)

Following the Atelier methodology , critical reflection is essential. Take time to analyze your work thoughtfully.

Reflection Questions

Write answers to these questions in a new file ( reflection.md ) or in comments at the bottom of your HTML:

1. Design Intent & Effectiveness (10 min)
  • Why did you choose these specific trends? What story or experience were you trying to create?
  • Do they enhance user understanding or just add visual appeal? Be honest!
  • What worked well? What aspects are you proud of?
  • What didn't work as expected? What would you change?
2. Accessibility & Inclusivity (10 min)
  • Who might struggle with your design choices? Consider users with motion sensitivity, visual impairments, or older devices.
  • Did you test with reduced motion preferences? How does your page adapt?
  • Is text readable? Check contrast ratios with WebAIM or Chrome DevTools.
  • Can users control the experience? (e.g., pause animations, toggle dark mode)
3. Technical Learning & Growth (10 min)
  • What new CSS properties or techniques did you learn?
  • What was most challenging? How did you solve it?
  • What resources did you use? (Demo files, MDN, StackOverflow, etc.)
  • If you were to do this again, what would you do differently?
  • How could you simplify your implementation? Could you achieve the same effect with less code?
